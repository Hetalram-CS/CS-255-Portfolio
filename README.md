# CS 255 Portfolio

## DriverPass Project Summary

The DriverPass project focused on designing a system for DriverPass, a company that helps students prepare for their driving tests. The client wanted a web-based system that allows customers to register, take online practice tests, schedule driving lessons, and access DMV training materials. The system also allows administrators, instructors, and staff to manage appointments, users, and reports.

## What I Did Well

I believe I did particularly well creating UML diagrams and translating the client’s requirements into a complete system design. I created use case, activity, sequence, and class diagrams that clearly represented the system functionality and user interactions. I also developed technical requirements that addressed security, scalability, and performance.

## What I Would Revise

If I could revise one part of my work, I would improve the UML class diagram by adding methods and additional relationships between classes. This would provide a more detailed representation of the system and make the design easier for developers to implement.

## Interpreting User Needs

I interpreted the user’s needs by carefully reviewing the DriverPass interview transcript and business requirements. I identified the features required by customers, instructors, administrators, and other users and incorporated them into my system design. Considering the user’s needs is important because a successful system must solve the client’s problems and provide a positive user experience.

## Software Design Approach

When designing software, I begin by gathering requirements and understanding the client’s goals. I use UML diagrams, process modeling, and object-oriented design techniques to visualize the system. In the future, I will continue using these strategies along with collaboration and documentation to create systems that are secure, reliable, and easy to maintain.
